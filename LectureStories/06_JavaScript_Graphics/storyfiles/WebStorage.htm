Client-side Storage

References

  • JavaScript: The Definitive Guide: Chapter 20. Client-Side Storage, Flanagan
  • Introduction to HTML5 Web Storage

Properties

  • Client-side storage serves to:
    • give the web browser a memory
    • store user preferences
    • store their complete state, so that they can resume exactly where you left off at the end of your last visit.
  • Client-side storage is
    • segregated by origin
    • pages from one site can’t read data stored by pages from another site
    • two pages from the same site can share storage and can use it as a communication mechanism.
    • data input in a form on one page can be displayed in a table on another page
  • Two kinds of web storage:
    • sessionstorage: temporary, e.g. only until the window closes or the browser exits
    • localStorage: permanent, so that it is available months or years later
