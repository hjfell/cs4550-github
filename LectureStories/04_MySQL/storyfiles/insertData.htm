Insert Data into a Table

mysql> INSERT into users
    -> (first_name, last_name, email, pass, registration_date)
    -> VALUES ('Harriet', 'Fell', 'me@universe.com', SHA1('mypass'),
    -> NOW());
Query OK, 1 row affected (0.24 sec)

SHA1( ) creates an encrypted string that is 40 characters long.
This is one-way encryption, not meant to be reversed.

NOW( ) returns the current time and date - on the server.

Inserting with just values and NULL for user_id.

mysql> INSERT INTO users VALUES
    -> (NULL, 'Doe', 'Jane', 'her@universe.com', SHA1('fido'), NOW());
Query OK, 1 row affected (0.55 sec)

Inserting several rows at once

mysql> INSERT INTO users (first_name, last_name, email, pass, registration_date) VALUES 
    -> ('John', 'Lennon', 'john@beatles.com', SHA1('Happin3ss'), NOW( )), 
    -> ('Paul', 'McCartney', 'paul@beatles.com', SHA1('letITbe'), NOW( )), 
    -> ('George', 'Harrison', 'george@beatles.com ', SHA1('something'), NOW( )), 
    -> ('Ringo', 'Starr', 'ringo@beatles.com', SHA1('thisboy'), NOW( ));
Query OK, 4 rows affected (0.00 sec)
Records: 4  Duplicates: 0  Warnings: 0

This is allowed in MySQL but not in other database applications.
