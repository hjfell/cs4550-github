Choosing Other Column Properties

NOT NULL
To force a field to have a value, add the NOT NULL description to the column type.
DECIMAL(6,2) NOT NULL

Indexes, Keys, and AUTO_INCREMENT

An index makes for easy retrieval but slower insertion.

A key is integral to normalization. There are primary and foreign keys. A primary key in one table may be a foreign key in another table.

Each table should have exactly one primary key that must obey:

  1. It must always have a value.
  2. The value must never change.
  3. The value must be unique for each record in the table.

In our users table, user_id will be the PRIMARY KEY.
MySQL will automatically index it.
AUTO_INCREMENT is also added to the column to to make the values unique.

Default Values

You can specify a default value
gender ENUM('M', 'F') default 'F'
If no default is specified, MySQL will choose some version of "zero".
