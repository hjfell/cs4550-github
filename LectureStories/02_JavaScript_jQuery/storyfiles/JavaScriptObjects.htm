JavaScript Objects

  • JavaScript’s fundamental datatype is the object.
  • An object is an unordered collection of properties
    • each of which has a name and a value
    • property names are strings
    • objects map strings to values
  • A JavaScript object inherits the properties of its “prototype.”
  • Objects have methods that are typically inherited properties
  • Any value in JavaScript that is not a string, a number, true, false, null, or undefined is an object.
  • Objects are mutable and are manipulated by reference rather than by value
  • If x refers to an object, and var y = x; is executed, then y holds a reference to the same object, not a copy of that object.
  • native objects: Arrays, functions, dates, and regular expressions
  • host objects: defined by the host environment. e.g. web browser, HTMLElement objects; may be native
  • user-defined objects: objects created by the execution of JavaScript code
  • own property is a property defined directly on an object
  • inherited property is a property defined by an object’s prototype object
