HTTP Cookies

A cookie, is usually a small piece of data sent from a website and stored in a user's web browser while a user is browsing a website. It contains some data:

  • A name-value pair containing the actual data
  • An expiry date after which it is trashed
    If you don't specify the expiry date the cookie is trashed when you close the browser.
    This expiry date should be in UTC (Greenwich) time.
  • The domain and path of the server it should be sent to

How Cookies Work

  • When a user requests a page from a server to which a cookie should be sent, the cookie is added to the HTTP header.
  • When the user's browser gets the page, the browser also gets the cookie, which it stores in a folder on your hard disk.
  • When the user requests a page from the site again, the browser looks on the local hard disk for a cookie associated with the URL. If the cookie exists, the browser sends the cookie to the site along with the page request.
  • Cookies are associated with a Web site, not with a specific page, so the browser and server will exchange the cookie information no matter what page you request from the site.
  • As the user visits different sites, each site might send a cookie to the user's browser as well; the browser stores all the cookies separately.
  • Most browsers support cookies of up to 4096 bytes. That is enough space for storing a few values on the user's computer. You might store a user number or other identifier so tha you could look up additional information in a database.
