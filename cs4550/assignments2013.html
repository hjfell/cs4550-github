January 15, 2013 		Getting Set Up
  • Set up MAMP, XAMPP, or some other Apache, MySQL, PHP server on your own machine.
  • Get an iPage account and domain name. You should make sure to get hosting for at least the length of this course (4 months). There are often sales but just in case there is not a good sale going, my account manager at iPage is . If you are registered for this course, you may contact him to sign up for hosting. He has promised a price of $42 for 12 months including a domain name for the year.
  • Set up an (S)FTP server so you can upload pages.
  • Get an editor for writing your HTML and other files. Notepad (PC) or TextEdit (Mac) will do but you will probably want an editor that does syntax highlighting and possibly checking. I use bbEdit on my MAC and textastic.app on my iPad but there are free alternatives
    • CoffeeCup
    • Komodo Edit
  • Find the directory where the Web server begins looking for files. Put a file there named "index.html" and fill it with text identifying this as your server. I suggest that you don't put your email address on the page for now to avoid spam.

    Connect to your server from a Web browser, using only the host name, "yoursite.com". (For the class site, this is cs4550.com but my index page is a forwarding page that takes you to a subdirectory so you actually end up at http://cs4550.com/cs4550/index.html.)

    Verify that http://yoursite.com shows your index page.

  • Use an HTML validator, e.g. http://validator.w3.org/, to make sure that your HTML is legal.
  • Set up a blog on a site of your choosing, e.g. wordpress.com, blogger.com, tumblr.com (or install wordpress on your own site but that is really not necessary). Write about your experiences in getting started.
  • Send me the information so I can find your site and your blog.
    NOTE: These will be public so that the whole class can view them.
For the next few weeks, you will each design and build a reservation/rental system. Your system should provide a way for you to learn and use a variety of web development tools.

Your system could be for reserving conference rooms at Northeastern University or ballrooms at the Ritz Carlton in Key Biscayne, Florida. It could be for for chartering sailboats on Cape Cod (Boat Ventures Case from is1500) or renting bicycles on Venice Beach.

January 22, 2013 		HTML, CSS, Designing a Reservation System
  • Reading:
    Get started reading the material on these first two sites. Some of you may know all of this, for others, the material may be new. We will discuss some of these constructs in class and learn more as we need to. The third page is a brief article on designing websites.
    • w3schools HTML
    • w3schools CSS
    • Planning a usable website: A three-step guide by Clara Teoh
  • Decide on the reservation/rental system that you would like to build.
    Plan the site for your system by filling in this template in your blog.
    • Background
      [Describe the background of your client and the main needs of the website; a couple of sentences is enough]
    • Requirements
      • Users
        [describe the users who will visit the website; what kinds of questions did you ask to come up with the users? Be sure to list those questions to show the client how to arrived at your list of users]
      • Use Cases
        [what will the users want to do on the site? How did you come up with those uses? Be sure to express your use cases in the format “[verb] [ action ] [ outcome ]”, e.g., “book an event room”; be sure to list different scenarios that must be considered when designing the website, for example, three-hour events, week-long events, specific locations, catering, etc.]
      • Functional Requirements
        [what specific needs do users have?]
      • Non-Functional Requirements
        [are there other considerations that are not listed above, e.g., mobile access, booking via smart phones?]
    • Design Specification
      • Content Needs
        [list the type of information that you will need to have on the website]
      • Media Needs
        [what kinds of media will you host on the site and why? Video? Images? Forms? Calendars? Blogs?]
      • External Links
        [what external websites will you link to? why]
  • Page Flow
    [using html and css, create an initial version of your site, e.g. Starship Pinafore Event Booking]

Friday
March 1, 2013 		Final Project Proposal
  • Use your blog to submit a proposal for your term project.
    For partners, one posting is enough. The other person should just post a pointer to their partner's blog.
  • Your proposal should include answers to the questions your were asked to write about for your reservation site.
  • Your term project should be a major work that should be at least as impressive as the reservation systems.
  • You may work in pairs or alone. We expect that 2-person projects reflect the efforts of two people and do more than solo projects. Let us know what each partner will be doing.
  • Grades will be given on the basis of look and feel, usability, quality of code, and general creativity.
  • Some general guidelines:
    • Make good use of style sheets and php includes to get a unified feel across your pages.
      There should be several pages. How many depends on the nature of your site.
    • There must be database interaction.
    • Forms should have validation.
    • You should use some API, e.g. google maps, rss feed, ...
    • For data-driven sites, use real data should be real, e.g. from Rotten Tomatoes or Amazon.
    • Include social media buttons.
    • Add some extras, e.g. a shopping cart, captchas, file uploads (of safe things like jpgs), cookies and/or local storage, session tracking ...
    • If some of these items do not apply to your site, let us know why they don't fit.
